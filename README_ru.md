## 📖 О проекте

**Manpupuner_42** — это демонстрация концепции (Proof of Concept) единого арбитра системных вызовов.

**Суть идеи:**  
Создать минимальную прослойку между железом и системными вызовами, которая понимает запросы из разных экосистем (POSIX и NT) и обрабатывает их через общий внутренний API из 7 базовых вызовов.

> **Это не эмуляция, не виртуализация и не второе ядро.**  
> Это — **унификация интерфейса** на самом низком уровне.

---

## 🏔️ Название

Проект назван в честь **Маньпупунёра** — геологического памятника на Северном Урале.

- **Семь каменных столбов-великанов** → символизируют **7 базовых системных вызовов**
- **Число 42** → универсальный ответ, поиск универсального решения

> *Семь столбов Маньпупунёра стоят веками.  
> Семь системных вызовов Manpupuner_42 могут стать основой для новой эры.*

🧱 Архитектура

Система состоит из трёх уровней:
text

┌─────────────────────────────────────────────────────────────┐
│                   ПРИЛОЖЕНИЯ / APPLICATIONS                  │
├──────────────────────────┬──────────────────────────────────┤
│  Слой A (POSIX-стиль)    │  Слой B (NT-стиль)              │
│  open(), read(), fork()  │  CreateFile(), ReadFile(), ...  │
├──────────────────────────┴──────────────────────────────────┤
│               ЕДИНЫЙ АРБИТР / UNIFIED ARBITER               │
│         7 базовых вызовов / 7 basic calls:                  │
│   READ_FILE, WRITE_FILE, CREATE_PROCESS, SLEEP,            │
│   ALLOC_MEMORY, FREE_MEMORY, LIST_FILES                    │
├─────────────────────────────────────────────────────────────┤
│               ИСПОЛНИТЕЛЬ (ЭМУЛЯЦИЯ) / EXECUTOR            │
│         Виртуальная ФС, память, процессы, время            │
└─────────────────────────────────────────────────────────────┘

🔧 7 базовых вызовов
№	Вызов	Описание
1	READ_FILE	Чтение данных из виртуальной ФС
2	WRITE_FILE	Запись данных в виртуальную ФС
3	CREATE_PROCESS	Создание нового процесса (эмуляция)
4	SLEEP	Приостановка выполнения
5	ALLOC_MEMORY	Выделение блока виртуальной памяти
6	FREE_MEMORY	Освобождение блока памяти
7	LIST_FILES	Список файлов в виртуальной ФС
💻 Пример использования
python

from hybrid_arbiter import HybridArbiter, LinuxCompatLayer, WindowsCompatLayer

# Создаём ядро-арбитр
kernel = HybridArbiter()

# Linux-стиль (POSIX)
linux = LinuxCompatLayer(kernel)
fd = linux.open("/home/user/hello.txt")
content = linux.read(fd, 50)
linux.close(fd)

# Windows-стиль (WinAPI)
windows = WindowsCompatLayer(kernel)
handle = windows.CreateFile("C:\\Users\\User\\hello.txt")
content = windows.ReadFile(handle, 50)
windows.CloseHandle(handle)

📊 Сравнение с аналогами
Подход	Принцип	Отличие от Manpupuner_42
Гибридное ядро	Одно ядро, одна экосистема	Не умеет запускать приложения из других систем
Трансляция (односторонняя)	A → B (только в одну сторону)	Только одно направление
Виртуализация	Два ядра, гипервизор	Требует виртуализации
Эмуляция API	Пользовательский режим	Работает на уровне API, а не ядра
Manpupuner_42	Одно ядро — два интерфейса	Двунаправленная совместимость на уровне ядра
